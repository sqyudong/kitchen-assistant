 # 厨房小秘书
 
 > 面向小型餐饮店、食堂档口、家庭厨房的通用食材库存管理工具。核心卖点是"说人话就能管库存"——在输入框中打字"今天买了5斤土豆放冰箱"，AI 自动完成结构化入库，无需学习任何表单操作。
 
 ---
 
 ## 功能特性
 
 - **AI 文本录入**：用自然语言描述入库/出库/移库操作，DeepSeek 自动解析执行
 - **多仓库多用户**：支持创建多个仓库，邀请员工协作管理
 - **角色权限体系**：老板（owner）拥有完整管理权，员工（staff）仅管理库存
 - **库存管理**：增删改查、位置筛选、过期预警（常温24h/48h黄红档，冷藏冷冻7天黄档）
 - **仓库位置**：自定义位置名称，创建时必选常温/冷藏/冷冻属性
 - **菜谱管理**：新增/删除菜谱，卡片式列表展示
 - **操作日志**：记录每次操作，删除类日志5分钟内可撤回
 - **成员管理**：邀请/移除员工，指定新老板
 - **删除撤销**：删除的库存项5分钟内可撤回
 - **管理员面板**：系统管理员可管理全部用户和仓库
 
 ## 技术栈
 
 | 层 | 技术 |
 |---|---|
 | 前端 | 原生 HTML5 + CSS3 + ES6 JavaScript，单文件 SPA |
 | 后端 | Supabase（PostgreSQL + REST API，通过 anon key + RLS 鉴权） |
 | AI | DeepSeek Chat API（deepseek-chat 模型，JSON 结构化输出） |
 | 存储 | 云端数据（Supabase）+ 本地仅存登录态/AI偏好/缓存 |
 | 部署 | Vercel（推荐）或任意静态托管平台 |
 
 ## 快速开始
 
 ### 前置准备
 
 1. 注册 Supabase → 创建项目 → 获取 Project URL + anon public key
 2. 注册 DeepSeek → 创建 API Key
 3. 选择部署平台（推荐 Vercel）
 
 ### 部署步骤
 
 1. **建表**：在 Supabase SQL Editor 中执行 init.sql，创建7张表 + 2个 RPC 函数 + RLS 策略
 2. **配置密钥**：打开 index.html，在开头的配置区填入你的 Supabase URL、anon key 和 DeepSeek API Key
 3. **部署**：将 index.html 拖入 Vercel 部署，或上传到任意静态托管服务
 4. **使用**：浏览器打开部署后的链接，注册账号 → 创建仓库 → 开始使用
 
 ### 手机端使用
 
 - 在浏览器中打开部署后的链接
 - iOS Safari：分享按钮 → 添加到主屏幕
 - Android Chrome：菜单 → 添加到主屏幕
